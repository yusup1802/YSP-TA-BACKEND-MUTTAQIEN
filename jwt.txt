🔒 Skenario A: Hanya accessToken yang Dihapus
Apa yang terjadi:

accessToken hilang, maka setiap request yang membutuhkan autentikasi (misalnya ke /dashboard) akan mendapatkan 403 Unauthorized.
Namun, karena ada refreshToken yang valid, interceptor akan mendeteksi error tersebut dan otomatis melakukan refresh token untuk mendapatkan accessToken baru.
Setelah mendapatkan token baru, request yang gagal sebelumnya akan dilanjutkan dengan token baru.

Kesimpulan: ✅ Masih bisa lanjut karena refresh token ada dan masih valid. Prosesnya berjalan dengan lancar berkat interceptor yang menangani proses refresh token.

🔒 Skenario B: Hanya refreshToken yang Dihapus
Apa yang terjadi:

Jika accessToken masih valid, maka aplikasi tetap bisa mengakses halaman yang memerlukan autentikasi, karena token tersebut masih ada.
Ketika accessToken kedaluwarsa, interceptor akan mencoba me-refresh token, namun gagal karena tidak ada lagi refreshToken.
Karena refresh token sudah tidak ada, maka aplikasi menghapus kedua token (akses dan refresh) dari localStorage dan mengalihkan pengguna ke halaman login.

Kesimpulan: ✅ User tetap login sementara tetapi setelah access token expired, aplikasi tidak bisa lagi me-refresh dan dipaksa logout karena kehilangan refresh token.

🔒 Skenario C: Keduanya (accessToken dan refreshToken) Dihapus
Apa yang terjadi:

Tanpa kedua token tersebut, setiap request yang memerlukan autentikasi akan gagal karena tidak ada Authorization header.
Tidak ada lagi refresh token untuk memperoleh access token baru, sehingga aplikasi tidak dapat melanjutkan request yang membutuhkan autentikasi.
Jika kamu menggunakan zustand untuk menyimpan status autentikasi (isAuthenticated), dan tidak menyimpan status ini secara persistent, maka ProtectedRoute akan langsung mengarahkan ke halaman login.

Kesimpulan: ❌ Tidak bisa akses apapun karena kedua token sudah hilang, dan aplikasi tidak bisa melanjutkan request apapun yang memerlukan autentikasi. Seolah-olah user logout secara paksa.

✅ Kesimpulan Umum:
Skenario A (Hanya accessToken hilang) — aplikasi akan tetap berjalan dan mendapatkan token baru menggunakan refreshToken.
Skenario B (Hanya refreshToken hilang) — akses akan tetap ada selama accessToken masih valid, tetapi jika token kedaluwarsa, user akan dipaksa logout.
Skenario C (Keduanya hilang) — aplikasi tidak bisa melanjutkan akses dan user akan langsung di-logout.
